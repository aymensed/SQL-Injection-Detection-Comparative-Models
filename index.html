What is TF-IDF & SVM? ü§ñ

Support Vector Machine (SVM) is a powerful classification algorithm that works by finding the optimal hyperplane that maximizes the margin between data classes.

Why SVM for SQLi? We use the **TF-IDF (Term Frequency-Inverse Document Frequency)** technique to convert text queries into numerical vectors, which the SVM model then uses to classify them as malicious or legitimate. This approach is highly effective for text classification.

Current Implementation: Our integrated SVM model uses a linear kernel with C=0.1 and processes queries through TF-IDF vectorization with 3000 maximum features.

Dataset Used üìä

The model was fine-tuned on a comprehensive dataset of 30,614 labeled queries, specifically designed for SQLi detection. The data is composed of:

  • **Label 0 (Normal)**: Standard SQL statements and benign user inputs.
  • **Label 1 (SQLi)**: Various modern SQL injection patterns including Tautologies and Error-based payloads.

Preprocessing: Input strings were tokenized, lowercased, and converted into features using the **TfidfVectorizer** object trained on our dataset. This is crucial for consistency.

Our Work and Methodology

Our work implemented a binary classification system using the **Support Vector Machine (SVM)** with a **Linear Kernel** for optimized performance in high-dimensional feature spaces.

Key Hyperparameters for SVM:

  • Feature Extraction: TF-IDF (max 3000 features)
  • Kernel: Linear
  • Regularization (C): 0.1
  • Data Split: 80% training, 20% testing

This approach ensured the model achieved high performance and reliable generalization capability.
